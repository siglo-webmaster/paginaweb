UC-00: Administrar Usuarios

Resumen: Los usuarios administradores pueden crear, modificar, borrar o cambiar clave de acceso de los usuario cliente
Prioridad: Esensial
Frecuencia de uso:
 Medio
Actores Directos:
Administradores
Principal escenario de Exito:
  1. Entrar al area de administracion->usuariosCliente
  2. tendra las opciones de creacion, actualizacion, borrado y cambio de clave para cada usuario cliente
  3. para crear usuarios selecciona crear nuevo, ingresar los datos correspondientes y salvar los cambios
  4. para cambiar password o actualizar datos del usuario seleccionar el boton correspondiente en cadacaso
  5. bara borrar usuario seleccionar el boton borrar y  seleccionar "si" cuando se le solicite confirmacion de borrado
Alternativas:
Escenarios extendidos:
  • El usuario cliente tiene la opcion de inscribirse en el formulario de creacion de cuenta nueva
  • el usuario cliente tiene la opcion de cambiar sus datos y su clave en su cuenta de usuario
Notas y preguntas

    UC-31: Sincronizacion de catalogo SHE

    Resumen: El sistema siglo sincronizara las actualizacionse sobre catalogo de productos cada vez que se presente un cambio en este, es decir cada vez que haya nuevos titulos o cada vez que se deje de vender algun producto,
    Prioridad: Esencial
    Frecuencia de uso:
    		 Muy Frecuente
    Actores Directos:
    Sistema Siglo
    Principal escenario de Exito:
    1. El sistema siglo cada vez que detecte cambios en el catalogo de productos, generara un archivo de texto con dichos cambios y lo depositara en el servidor de sincronizacion SIGLO - SHEWEB (VENUS)
    2. El sistemas SHEWEB revisa cada 5 minutos la presencia o no de archivos que reporten cambos en el catalogo
    3. El sistema SHEWEB procesa los cambios en el catalogo dejandolos disponibles inmediatamente en la pagina web
    Alternativas:

    UC-33: Sincronizacion de pedidos y pagos en linea

    Resumen: El sistema SHEWEB tiene la habilidad de conectarse al sistema siglo mediante a trvez del servidor de sincronizacion SIGLO -SHEWEB (Venus)  registrando pagos en linea y generando rutas de los productos vendidos en linea
    Prioridad: Esencial
    Frecuencia de uso:
    		 Medio
    Actores Directos:
    Sistema Siglo, Sistema SHEWEB, Servidor de Sincronizacion Siglo-SHEWEB
    Principal escenario de Exito:
    1. Una vez que el cliente haya echo el pago y el sistema SHEWEB  es  informado mediante el cambio de estado del  pedido
      2.  ben sea de forma manual o automatica mediante la respuesta del proveedor de pagos en linea.
    2. El cambio de estado del pedido genera que el sistema web se conecte con el servidor de sincronizacion SHE-WEB (VENUS) y reporte al sistema siglo del pago y de la nesecidad de generar una ruta a partir de los datos del pedido
    3. El sistema siglo genera una ruta automatica
    4. El sistema siglo reporta a la pagina web la generacion de la ruta, el numero de ruta y el estado de la misma.
    5. La pagina web actualiza el estado del pedido con los datos obtenidos desde el sistema siglo y los presenta al cliente en su cuenta de usuario
    6. Si el pedido incluia libros elecronicos el sistema SHEWEB enviara al cliente el correo con el enlace de descarga del libro
    7. si el pedido incluye un libro que debe ser tratado mediante impresion por demanda (PDO), generara el pedido de dicho libro hacia el proveedor de PDO
